Estudo Explicado de Python Django

1. Criação do Projeto e Aplicativo Django

  1. Criar um novo projeto Django: 
    django-admin startproject setup .
  2. Criar um novo aplicativo Django: 
    python manage.py startapp myapp
  3. Configurar URLs do aplicativo:

    No arquivo setup/urls.py, inclua as rotas do aplicativo:

    # setup/urls.py

from django.contrib import admin
from django.urls import include, path

urlpatterns = [
    path('admin/', admin.site.urls),
    path('myapp/', include('myapp.urls')),
]

    No arquivo myapp/urls.py, adicione as rotas do aplicativo:

    # myapp/urls.py

from django.urls import path
from . import views

urlpatterns = [
    path('', views.index, name='index'),
    # Adicione outras rotas aqui
]
  4. Aplicar migrações iniciais: 
    python manage.py makemigrations
python manage.py migrate

2. Configuração do Ambiente Virtual com virtualenv

  1. Instalar virtualenv: 
    pip install virtualenv
  2. Criar um ambiente virtual:

    No diretório raiz do seu projeto, execute:

    virtualenv .venv
  3. Ativar o ambiente virtual:
    • No Windows: 
      .\\.venv\\Scripts\\activate
    • No macOS/Linux: 
      source .venv/bin/activate
  4. Instalar Django no ambiente virtual: 
    pip install django python-dotenv

3. Uso do dotenv para Variáveis de Ambiente

  1. Crie um arquivo .env na raiz do seu projeto e adicione suas variáveis de ambiente:
    2. SECRET_KEY=your_secret_key
DEBUG=True
  2. Carregue as variáveis de ambiente no arquivo settings.py: 
    # setup/settings.py

import os
from dotenv import load_dotenv

load_dotenv()

SECRET_KEY = os.getenv('SECRET_KEY')
DEBUG = os.getenv('DEBUG', 'False') == 'True'

4. Configuração do Git

  1. Inicializar um repositório Git: 
    git init
  2. Criar um arquivo .gitignore na raiz do seu projeto e adicionar os seguintes conteúdos:
    3. # .gitignore

# Python
*.pyc
__pycache__/

# Django
db.sqlite3
*.log

# Virtualenv
.venv/

# Environment variables
.env

# Other
.DS_Store
  3. Adicionar arquivos ao repositório: 
    git add .
git commit -m "Initial commit"
  4. Adicionar um repositório remoto: 
    git remote add origin [url_https]
  5. Enviar o código para o repositório remoto: 
    git push -u origin master

Boas Práticas na Escrita de Código Python

  • PEP 8: Siga as convenções de estilo de código Python definidas na PEP 8.
    • Use indentação de 4 espaços.
    • Limite cada linha a um máximo de 79 caracteres.
    • Use espaços ao redor de operadores e após vírgulas.
    • Use linhas em branco para separar funções e classes, e grandes blocos de código dentro das funções.
  • Nomenclatura:
    • Use nomes descritivos e significativos para variáveis, funções, classes e métodos.
    • Nomes de classes devem estar em CamelCase.
    • Nomes de variáveis e funções devem estar em snake_case.
  • Documentação:
    • Documente suas funções e classes usando docstrings.
    • Adicione comentários explicativos onde necessário.
  • Organização de Pastas:
    • Separe suas aplicações Django em diretórios distintos dentro do projeto.
    • Use o diretório static para arquivos estáticos (CSS, JavaScript, imagens) e templates para arquivos de template HTML.
